# How To Use This Vault

This vault stores understanding, not canonical implementation truth.

## Use The Vault For

- architecture interpretation
- concept definitions
- decision rationale
- tensions and contradictions
- emergent patterns
- interpretation drift
- counterfactuals
- redacted evidence
- source digests

## Do Not Use The Vault For

- raw behavioral data
- secrets, tokens, or private URLs
- full copies of repo docs
- canonical schemas duplicated from code
- unredacted alpha-user details
- stale task lists
- model claims presented as facts

## Promotion Rule

Material starts in [[90_Inbox]]. It becomes durable only when it has source refs, links, uncertainty labels, and a clear relation to a concept, tension, pattern, decision, hypothesis, or domain.
