# Observer Sovereignty Watchlist

## Summary

This note parks a possible future failure mode: Lyra may reduce useful
self-monitoring burden, but must not become the user's source of meaning,
self-recognition, or final self-interpretation.

This is not doctrine, not a runtime contract, and not implementation authority.
It exists only as a watchlist item until product evidence shows the risk is
material.

## Core Tension

Lyra is designed to absorb some burdens currently carried by the user:

- remembering timestamps;
- reconstructing stale sessions;
- noticing pauses and drift;
- summarizing execution traces;
- estimating execution and occupancy windows;
- offering recovery prompts.

Those are bookkeeping and continuity burdens. They are reasonable software
targets.

The unresolved boundary is not generic "AI influence anxiety" or simple
over-trust. The sharper risk is:

```text
external interpretive compression replacing internal meta-cognition
```

A user may move from:

```text
What do I think about my behavior?
```

to:

```text
What does Lyra say my behavior is?
```

The failure mode is not that Lyra offers insight. It is that Lyra becomes the
default interpreter such that the user stops generating rival explanations.

Keep three states separate:

1. **Insight** - Lyra reflects a pattern the user did not notice.
2. **Authority transfer** - the user treats Lyra as more correct than their own
   self-perception.
3. **Meta-cognitive substitution** - the user no longer generates internal
   models without Lyra as the default interpreter.

Only the third state is the core watchlist risk.

The unresolved boundary remains:

```text
Lyra may reduce self-monitoring load.
Lyra must not become the user's source of meaning or final self-recognition.
```

The human may delegate ledger work. The human must retain interpretive
sovereignty.

## Not Currently Promoted

Do not promote this into doctrine yet.

Reason:

- no observed cohort evidence currently shows users deferring meaning,
  self-recognition, or agency to Lyra outputs;
- existing doctrine already covers adjacent risks: conscious influence,
  agency preservation, low-authority copy, provenance, reversibility, and
  dependence avoidance;
- premature doctrine would add conceptual overhead without a grounded product
  pressure.

## Plausibility Conditions

This risk becomes structurally plausible when all three conditions hold:

1. **High-coherence output** - Lyra gives consistent, structured interpretations.
2. **High-frequency exposure** - users interact repeatedly under stress or
   decision pressure.
3. **Action-utility loop** - Lyra outputs reliably reduce uncertainty or improve
   execution.

If all three are true, some users may outsource interpretation because it is
cognitively cheaper and more stable, not because they are irrationally
"over-trusting" the system.

This is not inherently bad. External meta-cognition can be valuable: journals,
therapists, coaches, dashboards, and analytics tools all partially externalize
reflection.

The boundary condition is:

```text
Does Lyra preserve the user's ability to disagree with it without cognitive
penalty?
```

## Watchlist Signals

Revisit this if trusted users say things like:

- "I don't know if my day was good until Lyra tells me."
- "Lyra understands me better than I do."
- "I wait for Lyra to decide what this pattern means."
- "I feel wrong when Lyra's interpretation disagrees with me."
- "I don't trust my own read of the day unless Lyra confirms it."

Also revisit if product behavior shows:

- users stop generating rival explanations;
- Lyra becomes the first-pass interpreter for identity rather than one
  hypothesis among several;
- disagreement feels like "Lyra versus me" instead of "two hypotheses";
- the user's self-model weakens in favor of the system-model;
- users stop making their own planning judgments without a Lyra prompt;
- users accept system interpretations despite explicit disagreement;
- users report anxiety when Lyra is unavailable;
- users ask Lyra to decide whether their effort, day, or recovery "counts";
- stronger mirrors increase compliance while reducing reported agency.

## Promotion Condition

Promote only if there is observed evidence that users defer self-recognition,
meaning, or agency to Lyra outputs.

Promotion should first update an existing doctrine or assumption register entry
rather than create a new subsystem.

If promoted, the strongest conceptual invariant is:

```text
Lyra must never be the only admissible explanation generator in the user's
cognition loop.
```

Possible future implication, not current doctrine: preserve alternative
interpretation space, explicitly support non-Lyra explanations, and treat
user-generated frames as first-class hypotheses rather than noise.

## Non-Goals

This note does not authorize:

- new tests;
- new UI;
- new analytics;
- new intervention gating;
- a new moral doctrine;
- stronger copy constraints beyond existing low-authority rules.

Until promoted, this remains a parked concept note.
