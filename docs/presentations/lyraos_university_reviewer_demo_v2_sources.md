## Claims Softened

- Baseet is described as a future/optional academic module unless a live seam is explicitly implemented.
- Provider activity is described as context/candidate evidence, not proof of learning, completion, or attention.
- Adaptive scheduling, AI synthesis, provider-progress inference, and session-continuity estimates are future-gated.
- Institution-facing outputs are described as aggregate pilot learning rather than individual student views.
- LyraSim is described as pre-scale stress testing, not proof of user acceptance or emotional safety.

## Language Choices

The deck avoids leading with defensive terms that can introduce reviewer anxiety. It uses positive access-boundary language instead: student workspace, confirmation-first workflow, aggregate pilot learning, and approved exports.
